Certificado já no emissor — ID não obtido automaticamente

O aplicativo confirmou que o certificado existe na conta do emissor fiscal, mas não conseguiu obter o identificador (ID) pelas consultas automáticas. Confira CNPJ, conta no painel web do emissor fiscal (onde o certificado foi registrado) e se URL base + API key do servidor são do mesmo ambiente.

Checklist

  1. CNPJ do formulário com 14 dígitos, igual ao certificado e ao cadastro no emissor.
  2. Mesma conta do emissor fiscal que a chave de API do backend.
  3. PLUGNOTAS_API_BASE_URL e PLUGNOTAS_API_KEY coerentes (sandbox/produção).
  4. Empresa existente na conta do emissor, se exigido para vincular certificado.

Documentação canônica no repositório: docs/operacao-mei-nfse.md (âncora #certificado-emissor-409-sem-id). Brief interno: pasta docs/brief/ no repositório (tema certificado no emissor, 409 sem id).
