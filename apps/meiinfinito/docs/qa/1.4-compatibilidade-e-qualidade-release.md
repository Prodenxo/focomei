## Riscos Residuais
1. Base frontend com erros de lint pre-existentes dificulta usar `npm run lint` como gate de bloqueio hoje.
2. Ausencia de scripts globais de `typecheck` e `test` impede validacao automatizada completa.
3. Regressao funcional completa de todo produto ainda depende de bateria manual dedicada (auth, transacoes, categorias e dashboard).

## Recomendacoes
1. Criar scripts de `typecheck` e `test` no workspace raiz.
2. Reduzir backlog de lint pre-existente por modulo para reabilitar gate de qualidade global.
3. Formalizar suite minima de regressao para fluxos MEI/NFSe.
